## 1. 시스템 개요

### 1-1. 소개

**일일 업무 보고 시스템**은 사원들이 매일 수행한 업무를 기록하고, 관리자가 팀 전체 현황을 한눈에 파악할 수 있도록 돕는 웹 기반 종합 업무 관리 도구입니다.

주요 기능은 다음과 같습니다:

- **일일 업무 보고서 작성 및 조회**: 날짜·부서·사원별 필터링 및 PDF 내보내기
- **부서별 AI 요약 자동 생성**: GPT-4.1 기반 당일 보고서 자동 요약
- **개인 업무 리포트**: 월별 또는 지정 기간 개인 성과 분석
- **프로젝트 관리**: 진행률·상태 추적 및 이슈 관리
- **미션/KPI 관리**: 팀·개인 목표 설정 및 달성률 추적
- **사원/부서/업체 관리**: 역할 기반 접근 제어

### 1-2. 역할 종류와 권한 요약

시스템에는 4가지 역할이 있으며, 각 역할마다 접근 가능한 기능이 다릅니다.

| 역할 | 표시명 | 배지 색상 | 설명 |
|------|--------|-----------|------|
| `operator` | 운영자 | 빨간색 | 시스템 전체 관리. 모든 업체·부서·사원 데이터에 접근 가능 |
| `company_manager` | 회사 관리자 | 보라색 | 자사 데이터 전체 관리. 부서 관리 가능 |
| `manager` | 부서 관리자 | 파란색 | 본인 부서 데이터 관리. 사원 추가 가능 |
| `user` | 사용자 | 회색 | 자사 데이터 조회, 본인 부서 보고서 작성 |

## 2. 시작하기

### 2-1. 로그인

시스템에 처음 접속하면 로그인 화면이 표시됩니다.

```
┌─────────────────────────────────────┐
│         업무 보고 시스템              │
│                                     │
│  이메일                              │
│  ┌───────────────────────────────┐  │
│  │ email@example.com             │  │
│  └───────────────────────────────┘  │
│                                     │
│  비밀번호                            │
│  ┌───────────────────────────────┐  │
│  │ ••••••••                      │  │
│  └───────────────────────────────┘  │
│                                     │
│  [         로그인         ]          │
└─────────────────────────────────────┘
```

**로그인 절차:**

1. **이메일** 입력란에 관리자로부터 부여받은 이메일 주소를 입력합니다.
2. **비밀번호** 입력란에 비밀번호를 입력합니다.
3. **로그인** 버튼을 클릭합니다.

**오류 메시지별 해결 방법:**

| 오류 메시지 | 원인 | 해결 방법 |
|-------------|------|-----------|
| 이메일 또는 비밀번호가 올바르지 않습니다 | 이메일 또는 비밀번호 오류 | 입력 정보 확인 후 재시도 |
| 로그인 처리 중 오류가 발생했습니다 | 서버 연결 오류 | 잠시 후 재시도하거나 관리자에게 문의 |
| 비밀번호를 잊어버린 경우 | — | 관리자(운영자 또는 부서 관리자)에게 비밀번호 초기화 요청 |

> ⚠️ **보안 주의**: 비밀번호는 최소 6자 이상이어야 하며, 첫 로그인 후 반드시 비밀번호를 변경하는 것을 권장합니다.

---

### 2-2. 화면 구성

로그인 후 화면은 크게 **헤더**, **네비게이션 바**, **메인 콘텐츠** 세 부분으로 구성됩니다.

```
┌────────────────────────────────────────────────────────────────┐
│  [회사명] ㅣ 업무 보고 시스템        홍길동 | [부서관리자]        │
│                                    [비밀번호 변경] [로그아웃]   │  ← 헤더
├────────────────────────────────────────────────────────────────┤
│  일일 업무 보고  업무 보고 작성  개인 업무 보고  프로젝트 관리   │
│  미션/KPI 관리  관리자                                          │  ← 네비게이션 바
├────────────────────────────────────────────────────────────────┤
│                                                                │
│                   메인 콘텐츠 영역                              │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

#### 헤더 구성 요소

- **회사명 / 업무 보고 시스템**: 클릭하면 메인 페이지(일일 업무 보고 조회)로 이동합니다.
- **사용자명**: 현재 로그인한 사원의 이름이 표시됩니다.
- **역할 배지**: 현재 역할이 색상 배지로 표시됩니다.
  - 빨간색 배지 = 운영자
  - 보라색 배지 = 회사 관리자
  - 파란색 배지 = 부서 관리자
  - 회색 배지 = 사용자
- **비밀번호 변경 버튼**: 클릭하면 비밀번호 변경 모달이 열립니다.
- **로그아웃 버튼**: 클릭하면 즉시 로그아웃되고 로그인 화면으로 이동합니다.

네비게이션 바에서 표시되는 메뉴는 역할에 따라 다릅니다.

| 메뉴 | 운영자 | 회사 관리자 | 부서 관리자 | 사용자 |

| 일일 업무 보고 | ✅ | ✅ | ✅ | ✅ |

| 개인 업무 보고 | ✅ | ✅ | ✅ | ✅ |
| 프로젝트 관리 | ✅ | ✅ | ✅ | ✅ |

| **관리자** | ✅ | ✅ | ✅ | ❌ |

## 3. 일일 업무 보고 조회 (`/`)

메인 페이지입니다. 특정 날짜의 전체 보고서를 부서별로 그룹화하여 표시합니다.

### 3-1. 필터 사용법

화면 상단에 다음 필터가 제공됩니다:

```
┌──────────────────────────────────────────────────────────────┐
│  날짜: [2026-03-03]   업체: [전체 ▼]   부서: [전체 ▼]        │
│  사원: [전체 ▼]                      [PDF 내보내기]           │
└──────────────────────────────────────────────────────────────┘
```

| 필터 | 설명 | 사용 가능 역할 |
|------|------|---------------|
| **날짜** | 조회할 날짜를 선택합니다 (기본값: 오늘) | 전체 |
| **업체** | 특정 업체만 필터링합니다 | 운영자만 표시 |
| **부서** | 특정 부서만 필터링합니다 | 운영자·회사 관리자 (부서 관리자·사용자는 본인 부서 고정) |
| **사원** | 특정 사원만 필터링합니다 | 전체 |

**필터 적용 방법:**
1. 드롭다운을 클릭하여 원하는 값을 선택합니다.
2. 선택 즉시 테이블이 자동으로 갱신됩니다.
3. `전체`를 선택하면 해당 필터가 초기화됩니다.

> ⚠️ **역할 제한**: 부서 관리자와 사용자는 부서 필터가 본인 부서로 자동 고정되며 변경할 수 없습니다.

---

### 3-2. 보고서 테이블 읽는 법

보고서는 부서별로 그룹화되어 표시됩니다.

```
┌─────────────────────────────────────────────────────────────────┐
│ ▶ [A업체 / 개발팀]                                               │
│   [AI 자동생성]  [직접작성/수정]                                  │
│   ┌───────────┐                                                  │
│   │ AI 요약 내용이 이 곳에 표시됩니다...                         │
│   └───────────┘                                                  │
├──────┬──────────┬────────────┬──────────┬──────────┬────────────┤
│ 사원 │   업무   │ 진행목표   │ 달성률   │  팀장평가│   비고     │
├──────┼──────────┼────────────┼──────────┼──────────┼────────────┤
│홍길동│ 기능 개발│ 70% 완료   │   80%    │ 우수     │            │
│      │ 코드 리뷰│ 리뷰 완료  │   100%   │ 양호     │            │
│김영희│ 연차     │     -      │    0%    │    -     │ 연차       │
└──────┴──────────┴────────────┴──────────┴──────────┴────────────┘
```

**부서 구분 행**: 각 부서의 상단에 `[업체명 / 부서명]` 형태로 표시됩니다.

**팀장평가**: 팀장이 직접 작성한 평가 코멘트가 텍스트로 표시됩니다.

**연차 처리**: 연차인 경우 업무 항목에 `연차`로 표시됩니다.

---

### 3-3. 보고서 수정·삭제

각 보고서 행 오른쪽에 **수정** 및 **삭제** 버튼이 있습니다.

> ⚠️ **역할 제한**: 본인이 작성한 보고서 및 본인 부서의 보고서만 수정·삭제할 수 있습니다. 운영자와 회사 관리자는 모든 보고서를 수정·삭제할 수 있습니다.

**보고서 수정 방법:**
1. 수정할 보고서 행의 **수정** 버튼을 클릭합니다.
2. 인라인 편집 폼이 나타납니다.
3. 내용을 변경한 후 **저장** 버튼을 클릭합니다.
4. 취소하려면 **취소** 버튼을 클릭합니다.

**보고서 삭제 방법:**
1. 삭제할 보고서 행의 **삭제** 버튼을 클릭합니다.
2. 확인 모달이 나타납니다.
3. **확인**을 클릭하면 삭제됩니다.

> ⚠️ **주의**: 삭제된 보고서는 복구할 수 없습니다. 신중하게 삭제하세요.

---

### 3-4. 부서별 AI 요약

각 부서 구분 행 아래에 AI 요약 영역이 있습니다.

```
┌─────────────────────────────────────────────────────────────────┐
│  ▶ [A업체 / 개발팀]                                              │
│                                                                  │
│  [AI 자동생성]  [직접작성/수정]                                  │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │  📝 요약이 없습니다. AI 자동생성 또는 직접 작성해 주세요. │   │
│  └──────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────┘
```

#### AI 자동생성 버튼

1. **[AI 자동생성]** 버튼을 클릭합니다.
2. 해당 날짜·부서의 보고서를 GPT-4.1이 자동으로 요약합니다.
3. 생성 중에는 버튼이 비활성화되고 로딩 표시가 나타납니다.
4. 완료 후 요약 내용이 해당 부서 영역에 표시됩니다.

> ⚠️ **역할 제한**:
> - 운영자: 모든 부서의 AI 요약 생성 가능
> - 회사 관리자: 자사의 모든 부서 요약 생성 가능
> - 부서 관리자·사용자: 본인 부서만 AI 요약 생성 가능

#### 직접작성/수정 버튼

1. **[직접작성/수정]** 버튼을 클릭합니다.
2. 텍스트 입력 영역이 나타납니다.
3. 요약 내용을 직접 입력하거나 기존 내용을 수정합니다.
4. **저장** 버튼을 클릭하면 저장됩니다.
5. 취소하려면 **취소** 버튼을 클릭합니다.

### 3-5. PDF 내보내기

현재 화면에 표시된 보고서를 PDF로 내보낼 수 있습니다.

1. 필터를 원하는 조건으로 설정합니다.
2. 화면 우측 상단의 **[PDF 내보내기]** 버튼을 클릭합니다.
3. PDF 생성이 완료되면 자동으로 다운로드됩니다.

> ℹ️ **참고**: PDF에는 현재 필터 조건에 맞는 보고서와 각 부서별 AI 요약 내용이 포함됩니다.

## 4. 업무 보고 작성 (`/create`)

사원들의 일일 업무 보고서를 작성하는 페이지입니다. 여러 사원의 보고서를 한 번에 작성하거나 수정할 수 있습니다.

### 4-1. 날짜 및 부서 선택

```
┌──────────────────────────────────────────────┐
│  보고 날짜                  부서 선택          │
│  ┌────────────────┐       ┌────────────────┐  │
│  │  2026-03-03    │       │  개발팀    ▼   │  │
│  └────────────────┘       └────────────────┘  │
└──────────────────────────────────────────────┘
```

1. **보고 날짜**: 날짜 입력 필드를 클릭하여 달력에서 날짜를 선택합니다. 기본값은 오늘 날짜입니다.
2. **부서 선택**:
   - 운영자·회사 관리자: 드롭다운에서 부서를 선택할 수 있습니다.
   - 부서 관리자·사용자: 본인 부서로 자동 고정되며 변경할 수 없습니다.

---

### 4-2. 사원 선택

부서를 선택하면 해당 부서의 사원 목록이 나타납니다.

```
┌──────────────────────────────────────────────────────────┐
│  사원명 선택 (2/5명)                   [전체 선택/해제]   │
│  ┌──────────────────────────────────────────────────────┐ │
│  │  ☑ 홍길동 (대리)  ☑ 김영희 (주임)  □ 이철수 (사원)  │ │
│  │  □ 박지연 (팀장)  □ 최민준 (사원)                    │ │
│  └──────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────┘
```

1. 보고서를 작성할 사원의 체크박스를 선택합니다.
2. **전체 선택** 버튼을 클릭하면 부서 내 모든 사원이 선택됩니다.
3. 이미 전체가 선택된 상태에서 버튼을 클릭하면 **전체 해제**됩니다.
4. 상단에 현재 선택된 사원 수와 전체 사원 수가 `(2/5명)` 형식으로 표시됩니다.

---

### 4-3. 기존 데이터 불러오기

사원을 선택하면 시스템이 해당 날짜에 이미 등록된 보고서가 있는지 자동으로 확인합니다.

**기존 데이터가 있는 경우:**
```
┌──────────────────────────────────────────────────────────┐
│  ℹ 기존 데이터 수정 중: 2026-03-03 날짜의 선택된 사원    │
│    보고서를 불러왔습니다.                                  │
└──────────────────────────────────────────────────────────┘
```

- 파란색 알림 박스가 표시되며 기존 데이터를 불러왔음을 알립니다.
- 폼에 기존 보고서 내용이 채워져 있습니다.
- **등록** 버튼 대신 **수정** 버튼이 표시됩니다.

### 5-1. 기간 선택

```
┌──────────────────────────────────────────────────────────────┐
```

### 7-5. KPI 달성률 자동 계산

KPI 달성률은 다음 공식으로 자동 계산됩니다:

```
달성률(%) = (현재값 / 목표값) × 100
```

- 목표값이 0이거나 비어 있으면 달성률은 0%로 표시됩니다.
- 달성률이 100%를 초과하는 경우 100%로 표시됩니다.

## 8. 관리자 페이지 (`/admin`)

사원, 부서, 업체, AI 프롬프트를 관리하는 페이지입니다.

> ⚠️ **접근 권한**: 운영자, 회사 관리자, 부서 관리자만 접근할 수 있습니다. 일반 사용자는 이 페이지에 접근할 수 없습니다.

관리자 페이지는 탭 구조로 이루어져 있습니다:

```
┌──────────────────────────────────────────────────────────────────┐
│  [사원 관리]  [부서 관리]  [업체 관리]  [프롬프트 관리]           │
│  (항상 표시) (operator,    (operator만) (operator만)              │
│             company_manager)                                     │
└──────────────────────────────────────────────────────────────────┘
```

| 탭 | 접근 가능 역할 |
|----|---------------|
| 사원 관리 | 운영자, 회사 관리자, 부서 관리자 |
| 부서 관리 | 운영자, 회사 관리자 |
| 업체 관리 | 운영자만 |
| 프롬프트 관리 | 운영자만 |

---

### 8-1. 사원 관리

#### 사원 목록 조회

사원 목록은 사원코드 기준으로 오름차순 정렬됩니다.

```
┌──────────────────────────────────────────────────────────────────────┐
│  사원 관리                                          [+ 사원 추가]    │
├──────────────────────────────────────────────────────────────────────┤
│  업체: [전체 ▼]  부서: [전체 ▼]  역할: [전체 ▼]                     │
│  검색 (사원명/이메일): [이름 또는 이메일 검색]   [필터 초기화]        │
│  총 25명                                                             │
├──────────┬──────────┬──────────┬──────────┬─────────┬──────┬────────┤
│ 사원코드 │ 사원명   │ 직책     │ 부서     │ 업체    │역할  │ 작업  │
├──────────┼──────────┼──────────┼──────────┼─────────┼──────┼────────┤
│ EMP001   │ 홍길동   │ 대리     │ 개발팀   │ A업체   │사용자│수정   │
│          │          │          │          │         │      │비밀번호│
│          │          │          │          │         │      │삭제   │
└──────────┴──────────┴──────────┴──────────┴─────────┴──────┴────────┘
```

**필터 옵션:**

| 필터 | 설명 | 사용 가능 역할 |
|------|------|---------------|
| **업체** | 업체별 필터링 | 운영자만 표시 |
| **부서** | 부서별 필터링 | 운영자, 회사 관리자 |
| **역할** | 역할별 필터링 | 전체 |
| **검색** | 사원명·이메일 검색 | 전체 |

---

#### 사원 추가

1. **[+ 사원 추가]** 버튼을 클릭합니다.
2. 다음 필드를 입력합니다:

| 필드 | 필수 | 설명 |
|------|------|------|
| **역할** | ✅ | 사원의 역할을 선택합니다 (사용자/부서 관리자/회사 관리자/운영자) |
| **업체** | ✅ (operator 제외) | 소속 업체를 선택합니다. 운영자 자신은 업체 소속 없음 |
| **부서** | ✅ (manager, user) | 소속 부서를 선택합니다. 업체 선택 후 해당 업체 부서 목록이 표시됩니다 |
| **사원코드** | ✅ | 고유 사원코드를 입력합니다 |
| **사원명** | ✅ | 사원 이름을 입력합니다 |
| **직책** | ✅ | 사원의 직책을 입력합니다 (예: 대리, 팀장) |
| **이메일** | — | 로그인에 사용할 이메일 주소를 입력합니다 |
| **초기 비밀번호** | — | 최소 6자 이상의 초기 비밀번호를 입력합니다 |

3. **추가** 버튼을 클릭합니다.

> ⚠️ **역할별 제한**:
> - 부서 관리자: 본인 부서로 고정. 회사 관리자·운영자 역할 선택 불가
> - 회사 관리자: 자사(본인 업체)에만 추가 가능
> - 운영자: 모든 업체에 모든 역할로 추가 가능

---

#### 사원 수정

1. 수정할 사원 행의 **수정** 버튼을 클릭합니다.
2. 폼에 기존 정보가 채워집니다.
3. 변경할 내용을 수정합니다.
4. **수정** 버튼을 클릭합니다.

> ℹ️ **참고**: 수정 시 비밀번호 필드를 비워두면 기존 비밀번호가 유지됩니다.

---

#### 사원 삭제

1. 삭제할 사원 행의 **삭제** 버튼을 클릭합니다.
2. `"[사원명] 사원을 삭제하시겠습니까?"` 확인 다이얼로그가 나타납니다.
3. **확인**을 클릭하면 삭제됩니다.

> ⚠️ **주의**: 사원 삭제 후에는 해당 사원의 로그인이 불가능해집니다. 삭제 전 충분히 검토하세요.

---

#### 비밀번호 재설정 (관리자용)

관리자가 특정 사원의 비밀번호를 강제로 재설정할 수 있습니다.

1. 해당 사원 행의 **비밀번호** 버튼을 클릭합니다.
2. 비밀번호 재설정 모달이 열립니다.
3. 새 비밀번호를 입력합니다 (최소 6자 이상).
4. **재설정** 버튼을 클릭합니다.

> ⚠️ **역할 제한**: 운영자는 모든 사원의 비밀번호를 재설정할 수 있습니다. 회사 관리자·부서 관리자도 이 버튼을 통해 자사/자부서 사원의 비밀번호를 재설정할 수 있습니다.

### 8-2. 부서 관리

> ⚠️ **접근 권한**: 운영자, 회사 관리자만 접근 가능합니다.

#### 부서 목록 조회

```
│  총 8개 부서                                                      │
├──────────────────────────────────┬──────────────┬────────────────┤

└──────────────────────────────────┴──────────────┴────────────────┘
```

#### 부서 추가

|------|------|------|

3. 부서명을 수정합니다.

3. **확인**을 클릭합니다.

> ⚠️ **주의**: 소속 사원이 있는 부서를 삭제하면 해당 사원들의 부서 정보가 유실될 수 있습니다. 부서 삭제 전 반드시 소속 사원을 다른 부서로 이동시키거나 삭제하세요.

## 9. 계정 관리

### 9-1. 비밀번호 변경

본인의 비밀번호를 변경하는 방법입니다.

1. 화면 우측 상단의 **[비밀번호 변경]** 버튼을 클릭합니다.
2. 비밀번호 변경 모달이 열립니다.

```
┌───────────────────────────────────────────────┐
│  비밀번호 변경                                 │
│                                               │
│  현재 비밀번호                                │
│  ┌─────────────────────────────────────────┐  │
│  │ ••••••••                                │  │
│  └─────────────────────────────────────────┘  │
│                                               │
│  새 비밀번호                                  │
│  ┌─────────────────────────────────────────┐  │
│  │ 새 비밀번호 (최소 6자)                  │  │
│  └─────────────────────────────────────────┘  │
│                                               │
│  새 비밀번호 확인                             │
│  ┌─────────────────────────────────────────┐  │
│  │ 새 비밀번호를 다시 입력하세요           │  │
│  └─────────────────────────────────────────┘  │
│                                               │
│              [취소]  [변경]                   │
└───────────────────────────────────────────────┘
```

3. **현재 비밀번호**를 입력합니다.
4. **새 비밀번호**를 입력합니다 (최소 6자 이상).
5. **새 비밀번호 확인**란에 새 비밀번호를 한 번 더 입력합니다.
6. **변경** 버튼을 클릭합니다.

**비밀번호 변경 규칙:**
- 새 비밀번호는 최소 **6자 이상**이어야 합니다.
- 현재 비밀번호와 다른 비밀번호를 설정해야 합니다.
- 새 비밀번호와 확인 입력이 일치해야 합니다.

**오류 메시지:**

| 오류 메시지 | 원인 |
|-------------|------|
| 모든 필드를 입력해주세요 | 빈 필드가 있는 경우 |
| 새 비밀번호는 최소 6자 이상이어야 합니다 | 새 비밀번호가 너무 짧은 경우 |
| 새 비밀번호가 일치하지 않습니다 | 확인 입력이 다른 경우 |
| 새 비밀번호는 현재 비밀번호와 달라야 합니다 | 동일한 비밀번호를 입력한 경우 |
| 현재 비밀번호가 올바르지 않습니다 | 현재 비밀번호 오류 |

---

### 9-2. 로그아웃

2. 즉시 로그아웃되고 로그인 화면으로 이동합니다.

> ℹ️ **참고**: 로그아웃 후 브라우저를 닫거나 세션이 만료되면 다시 로그인해야 합니다.

## 11. 자주 묻는 질문 (FAQ)

### Q1. 비밀번호를 잊어버렸을 때 어떻게 하나요?

**A.** 스스로 비밀번호를 초기화할 수 없습니다. 소속 팀의 **부서 관리자, 회사 관리자, 또는 운영자**에게 비밀번호 초기화를 요청하세요.

관리자는 다음 경로에서 초기화할 수 있습니다:
> 관리자 페이지(`/admin`) → 사원 관리 탭 → 해당 사원 행의 **[비밀번호]** 버튼 클릭

초기화 후 제공받은 임시 비밀번호로 로그인한 다음, **[비밀번호 변경]** 버튼을 통해 즉시 새 비밀번호로 변경하는 것을 권장합니다.

---

### Q2. 업체 필터가 보이지 않습니다.

**A.** 업체 필터는 **운영자 역할**에게만 표시됩니다.

- 회사 관리자·부서 관리자·사용자는 자신의 소속 업체 데이터만 조회할 수 있어 업체 필터가 불필요합니다.
- 본인이 운영자 역할인데도 업체 필터가 보이지 않는다면, 브라우저를 새로고침(`F5`)하거나 재로그인해 보세요.

---

### Q3. AI 요약 생성 시 오류가 발생합니다.

**A.** 다음 사항을 순서대로 확인하세요:

1. **해당 날짜·부서에 보고서가 존재하는지 확인**
   - AI 요약은 당일 등록된 보고서를 기반으로 생성됩니다. 보고서가 없으면 생성할 수 없습니다.

2. **역할 및 부서 확인**
   - 부서 관리자·사용자는 본인 부서만 AI 요약을 생성할 수 있습니다. 다른 부서의 AI 자동생성 버튼을 클릭하면 오류가 발생합니다.

3. **잠시 후 재시도**
   - AI 서버의 일시적인 과부하로 실패할 수 있습니다. 30초 후 다시 시도해 보세요.

4. **지속적으로 오류가 발생하는 경우**
   - 시스템 관리자(운영자)에게 문의하세요. OpenAI API 키 설정 또는 데이터베이스 마이그레이션 미적용으로 인한 서버 오류일 수 있습니다.

---

### Q4. 보고서 목록에 보고서가 보이지 않습니다.

**A.** 다음 사항을 확인하세요:

1. **날짜 필터 확인**: 조회하려는 날짜로 날짜 필터가 설정되어 있는지 확인하세요. 기본값은 오늘 날짜입니다.

2. **부서 필터 확인**: 부서 필터가 `전체`로 설정되어 있는지 확인하세요 (운영자·회사 관리자인 경우).

3. **역할 및 부서 확인**: 부서 관리자·사용자는 본인 부서의 보고서만 볼 수 있습니다. 다른 부서의 보고서는 조회되지 않습니다.

4. **보고서 미등록 확인**: 해당 날짜에 보고서가 실제로 등록되어 있는지 확인하세요. 보고서 작성 페이지(`/create`)에서 등록 여부를 확인할 수 있습니다.

---

### Q5. 부서 변경 후 사원 목록이 비어 있습니다.

**A.** 보고서 작성 페이지에서 부서를 변경하면 사원 선택이 자동으로 초기화됩니다. 정상적인 동작입니다. 새로운 부서의 사원 목록이 표시될 때까지 잠시 기다린 후 사원을 다시 선택하세요.

---

### Q6. 미션/KPI 달성률이 잘못 계산되는 것 같습니다.

**A.** KPI 달성률은 `(현재값 / 목표값) × 100`으로 자동 계산됩니다.

- 현재값 또는 목표값을 올바르게 입력했는지 확인하세요.
- 목표값이 `0`이거나 비어 있으면 달성률은 `0%`로 표시됩니다.
- 달성률이 100%를 초과하는 경우 100%로 제한됩니다.

---

### Q7. PDF 내보내기가 되지 않습니다.

**A.** 다음 사항을 확인하세요:

1. **팝업 차단 해제**: 브라우저의 팝업 차단 설정이 이 사이트를 차단하고 있을 수 있습니다. 브라우저 주소창 우측의 팝업 차단 아이콘을 클릭하여 허용하세요.

2. **보고서 데이터 확인**: 화면에 표시된 보고서 데이터가 있는지 확인하세요. 보고서가 없으면 PDF가 생성되지 않을 수 있습니다.

3. **잠시 후 재시도**: 서버 부하로 인해 일시적으로 느릴 수 있습니다. PDF 내보내기 버튼 클릭 후 30초 이상 기다려도 반응이 없으면 다시 시도하세요.

---

### Q8. 사원 추가 시 사원코드 중복 오류가 발생합니다.

**A.** 사원코드는 동일한 업체 내에서 고유해야 합니다. 이미 사용 중인 사원코드를 입력한 경우 발생합니다. 다른 사원코드를 입력하거나, 기존 사원 목록에서 해당 코드가 실제로 사용 중인지 확인하세요.

---

### Q9. 로그인이 계속 풀립니다.

**A.** 로그인 세션은 JWT 토큰 기반으로 유지되며, 토큰 만료 시 자동으로 로그아웃됩니다. 정기적으로 재로그인하거나, 세션이 자주 만료된다면 시스템 관리자에게 JWT 만료 시간 설정을 문의하세요.

---

### Q10. 관리자 페이지에서 특정 탭이 보이지 않습니다.

**A.** 관리자 페이지의 탭은 역할에 따라 다르게 표시됩니다:

- **부서 관리 탭**: 운영자·회사 관리자만 표시됩니다.
- **업체 관리 탭**: 운영자만 표시됩니다.
- **프롬프트 관리 탭**: 운영자만 표시됩니다.

필요한 기능에 접근하려면 더 높은 권한의 역할이 필요합니다. 운영자에게 역할 변경을 요청하세요.
